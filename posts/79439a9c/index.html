官方案例

CounterApp

IModel

继承IModel,绑定Count 数据

ISystem

引入成就系统

IUtility

unity相关的接口

CounterApp

初始化以上自定义的接口

IController

获取自定义Model,view绑定命令接口, GetArchitecture 注册App中的接口,因为QFramework中需要从mArchitecture中返回自定义的IModel,ISystem,IUtility等接口

IOCContainerExample

IOC容器解释

PointPointPoint

文件结构

Command

命令模式中,主要在改变GameModel中绑定的BindableProperty变量,作为一种桥接,传递View->Model的改变

Event

定义事件,通过RegisterEvent在界面和系统注册,在符合条件的发送SendEvent,就可以在游戏中改变注册的泛型方法

Game

全部继承了IController,为游戏中的实体,处理游戏中实体部分的表现,通过命令模式发送数据

Model

只记录了全局游戏相关的数据,和上个案例使用了相同的方法绑定数据

System

继承ISystem,在各自系统定义各自系统的数据,OnInit初始化注册事件

PointGame.cs

注册全部的Modle和System,QFramework的要求

ShootingEditor2D

相较于上两个案例更加复杂,添加了相机和关卡编辑,不在使用单个场景,使用Resource加载prefab
